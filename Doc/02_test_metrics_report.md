## 5. 처리 방식 분석

### 5.1 규칙 기반 vs LLM 폴백

| 처리 방식 | 건수 | 비율 | 평균 시간 |
|----------|------|------|----------|
| 규칙 기반 (Fast) | 8,500 | 85.0% | 95ms |
| LLM 폴백 (Accurate) | 1,350 | 13.5% | 450ms |
| 수동 처리 필요 | 150 | 1.5% | - |

### 5.2 LLM 폴백 트리거 조건

1. **복잡한 주소 형식** (45%)
   - 신규 도로명주소 + 상세주소 조합
   - 아파트 동/호수 비표준 표기

2. **비표준 상품 표현** (30%)
   - "그거", "저번에 산 것" 등 지시어
   - 상품명 축약/변형

3. **복합 요청사항** (25%)
   - 다중 조건 요청
   - 문맥 의존적 표현
